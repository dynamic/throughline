Why throughline

Most memory tools for AI agents replay the conversation. throughline captures the work and the state, separates cheap automatic capture from deliberate curated handoff, and binds into Claude Code's native memory.

Why not a manual HANDOFF.md?

You have to remember to write it, and you write it from memory, which after a long session is itself reconstructed from a compacted transcript. throughline captures continuously and distills from a buffer that does not forget.

Why not CLAUDE.md?

That is for durable preferences and conventions, the things true every session. throughline is for per-session work state: what you did today, where you stopped, what is still open.

Why not native /memory?

Native memory holds global facts. throughline holds project work state and live git/PR context, and it promotes the genuinely durable facts up into native memory rather than competing with it.

How it works

Three layers, each doing what it is actually capable of.

01 / CAPTURE

Continuous capture

PostToolUse hook

Appends a structured one-liner per mutating action to a per-session buffer: the command or file, flagged if it was interrupted, with obvious secrets masked first. Mechanical and cheap. You never see it; it just protects you.

02 / HANDOFF

Judged handoff

handoff skill

At wrap-up the agent distills the buffer plus context into a durable HANDOFF.md and a timestamped log, and promotes durable facts into native memory. Judgment, so it is a skill, not a hook.

03 / SAFETY NET

Safety-net flush

SessionEnd hook

Stamps the buffer on exit, so a session that ended without a handoff is surfaced next time for retroactive distillation. Nothing is ever silently lost.

🧷

Built to survive compaction

When Claude Code compacts a long conversation, the transcript is summarized and detail is lost. The action buffer does not care: it lives on disk, appended after every action, independent of the context window. A PreCompact hook stamps a boundary marker at the seam, and SessionStart re-fires on compact to point the agent back at the surviving buffer. Honest scope: the what survives on disk; the why lives in the conversation, so run a handoff before a long session compacts to keep the reasoning too.

Install and configure

No build step, no account, no server. It is hooks and skills, all local.

1. Install the plugin

# in Claude Code
/plugin marketplace add dynamic/throughline
/plugin install throughline@throughline
# then reload or restart the session
/reload-plugins

2. Choose where state lives

# default: .claude/throughline/ per project
# or unify with a portable .agent/ convention
export THROUGHLINE_DATA_DIR=.agent/handoff

3. Keep it out of git (default)

# in your project's .gitignore
.claude/throughline/

4. Reinforce auto-handoff (optional)

# add to project or global CLAUDE.md
When a session winds down, run the
handoff skill and report
the diff, don't wait to be asked.

throughline activates automatically the first time any hook fires in a project - no manual opt-in step. To keep it out of a specific project, drop an empty .throughlineignore file at the project root. All of it - HANDOFF.md, logs/, buffer/ - is per-operator working memory and stays local by default; committing HANDOFF.md/logs/ so teammates and fresh clones get oriented is a deliberate opt-in (see the README's "Local by default"), not the default. Requires git and jq on your PATH; without jq, capture cannot run and the SessionStart block says so instead of failing silently.
